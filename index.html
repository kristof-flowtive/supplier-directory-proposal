Architecture

Automation Flow

Four scenarios cover the entire directory. Each one runs through Softr against your live Airtable base.

Scenario 1 · Public Directory Browse

Visitor lands → Softr directory page → Airtable (approved suppliers) → Filtered results

Scenario 2 · Vendor Submission

Vendor submits form → Softr form block → Airtable record (status=Pending) → Admin notification

Scenario 3 · Admin Approval

You review → Softr admin view → Airtable (status=Live) → Profile goes public

Scenario 4 · Buyer Inquiry

Visitor clicks Contact → Softr inquiry form → Make.com routing → Inbox + Airtable log
